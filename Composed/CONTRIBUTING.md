# Contributing to iFoundation

Thank you for your interest in contributing! This document covers our development workflow, code style, and review process.



## Code Style

- SwiftFormat configuration in `.swiftformat`
- SwiftLint with custom rules in `.swiftlint.yml`
- 120 character line limit
- Explicit self where required for clarity
- Protocol-oriented design
- Actor isolation for thread safety



## Pull Request Process

1. Branch from `main` or `develop`
2. Make changes with tests
3. Run pre-commit hooks
4. Open PR with description
5. CI must pass
6. Code review required
7. Squash and merge



## Code of Conduct

Be respectful, inclusive, and constructive. Focus on the code, not the person.
